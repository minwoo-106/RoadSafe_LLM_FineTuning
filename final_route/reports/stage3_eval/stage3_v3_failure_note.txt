[Stage3 v3 실험 요약]

목적:
- Stage3 v2의 키워드 회수율은 유지하면서 END 출력과 답변 형식 안정성을 개선한다.
- 답변 길이를 줄이고, 섹션 누락과 조기 종료 문제를 줄이는 방향으로 데이터와 학습 스크립트를 수정한다.

변경 내용:
- roadsafe_stage3_user_qa_v3 데이터 생성 스크립트 추가
- split_v3 생성 스크립트 추가
- Stage3 v3 sanity/full 학습 스크립트 추가
- v3 golden eval 및 promptfix eval 스크립트 추가
- v2 데이터 기반으로 hard case 일부를 추가하고, <END> 출력과 섹션 순서 지시를 강화함

결과:
- END 출력은 40/40으로 개선됨
- forbidden 표현은 40/40 통과
- keyword recall은 v2 수준을 유지함
- 그러나 섹션 제목 변형과 한국어 깨짐이 증가함
- section_pass 0/40으로 최종 후보에서 제외함

원인 추정:
- 작은 데이터셋에 learning_rate 2e-4, epoch 3이 강하게 작용했을 가능성이 있음
- 답변을 짧게 쓰라는 지시가 조기 종료와 압축 답변 성향을 강화했을 가능성이 있음
- 섹션명과 한국어 표기 안정성이 붕괴됨

후속 방향:
- Stage3 v4는 v2 데이터 구조를 기반으로 한다.
- v3 hard case는 일부만 보강한다.
- learning_rate를 낮추고 epoch를 줄인다.
- 정확한 섹션명과 <END> 위치를 함께 보강한다.
